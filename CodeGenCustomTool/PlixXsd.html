Elements  (Complex Types, Simple Types)

  • alternateBranch (alternateBranchType): An alternate branch for a conditional branch block. (C#: else if, VB: ElseIf)
  • arrayBound (arrayBoundType): Bound information to specify something other than a 0-based unsized array.
  • arrayDescriptor (arrayDescriptorType): Define an array other than a simple array. Nested arrays are defined by nesting array descriptors.
  • arrayInitializer (arrayInitializerType): Assign initial values to an array
  • assign (assignType): Assignment statement.
  • attachEvent (attachEventType): Attach an event procedure to an event. The left hand side will generally be a callInstance, callThis, or callStatic with type='event', and the right hand size will generally be a delegate instance; or a callInstance, callThis, or callStatic with type='MethodReference'.
  • attribute (attributeType): An attribute declaration. Attached to members and types.
  • autoDispose (autoDisposeType): The definition of a disposable variable and the associated body code. (C#: using, VB: Using)
  • beforeLoop (beforeLoopType): A statement to execute after each pass through a loop.
  • binaryOperator (binaryOperatorType): A binary operator with left and right elements.
  • blockLeadingInfo (blockLeadingInfoType): Non-code elements immediately preceding secondary code blocks (alternateBranch, fallbackBranch, catch, fallbackCatch, finally, case, fallbackCase). Comments and pragma conditions for primary code blocks (branch, lock, loop, switch, try) can be added to the container code body.
  • blockTrailingInfo (blockTrailingInfoType): Non-code elements immediately after secondary code blocks (alternateBranch, fallbackBranch, catch, fallbackCatch, finally, case, fallbackCase). Comments and pragma conditions for primary code blocks (branch, lock, loop, switch, try) can be added to the container code body.
  • branch (branchType): A conditional branch block. (C#: if, VB: If)
  • break (breakType): Break out of the enclosing construct.
  • callInstance (callInstanceType): A call to any instance element on a class with a specified type. Note that callThis should be used to access elements on the current class.
  • callNew (callNewType): Create a new element of the specified type. If the type is an array, then this will create an array.
  • callObject (expressionContainerType): The calling object for a callInstance expression.
  • callStatic (callStaticType): A call to any static element on a class with a specified type. Note that callThis can be used to use an unqualified static member on a local class.
  • callThis (callThisType): A call to any instance or static element on this class or a base class.
  • case (caseType): One or more condition expressions to match against the condition element of the containing switch element. Note that multiple conditions are allowed. It is the responsibility of the individual plix language formatters to ensure that non-empty case statements do not fallthrough. Empty cases should be specified as multiple condition elements on a single case.
  • cast (castType): Cast the expression to the specified type.
  • catch (catchType): The definition of a catch variable and its body code.
  • class (classType): A class definition. Contains nested type definitions, fields, and procedure definitions. (C#:class, VB:Class)
  • comment (commentType): A comment.
  • condition (expressionContainerType): The test expression for a branch, alternateBranch, loop, switch, case, or conditionalOperator.
  • conditionalOperator (conditionalOperatorType): A conditional operator with condition, left, and right elements. Only valid inside an inlineStatement element.
  • continue (continueType): Continue with the next element of the enclosing loop or iterator construct.
  • decrement (decrementType): Decrement the expression by one. The expression will generally be a nameRef or a call* element with type='field'.
  • defaultValueOf (defaultValueOfType): The default value for a given type. (C#: default(), VB: Nothing)
  • delegate (delegateType): A delegate definition. Contains parameter definitions. (C#:delegate, VB:Delegate)
  • derivesFromClass (derivesFromClassType): Specifies a base class for another class. Used as a child of the Class element. The implementsInterface element should be used for the base of an interface element and to specify additional implemented interfaces on a class.
  • detachEvent (detachEventType): Detach an event procedure from an event. The left hand side will generally be a callInstance, callThis, or callStatic with type='event', and the right hand size will generally be a callInstance, callThis, or callStatic with type='MethodReference'.
  • directTypeReference (directTypeReferenceType): A direct use of a type in an expression. Used for typeEquality and typeInequality binary operators.
  • docComment (docCommentType): Documentation comments, allowed on members
  • enum (enumType): An enumeration definition. Contains enumItem definitions. (C#:enum, VB:Enum)
  • enumItem (enumItemType): An item in an enumeration
  • event (eventType): Represents an event definition.
  • explicitDelegateType (explicitDelegateTypeType): An explicit delegate to use as the type for an event. If the delegate type is not provided, then some languages may generate an implicit delegate type based on the event's signature definition.
  • expression (expressionType): An explicit expression element to contain and modify any other expression elements.
  • fallbackBranch (secondaryCodeBodyType): The unconditional fallback portion of a branch construct (C# else, VB: Else)
  • fallbackCase (secondaryCodeBodyType): The unconditional fallback case for a switch construct (C# default, VB: Case Else)
  • fallbackCatch (secondaryCodeBodyType): The untyped fallback catch for a try construct (C# catch, VB: Catch)
  • falseKeyword (keywordType): Keyword representing the boolean false value.
  • field (fieldType): An element representing a field
  • finally (secondaryCodeBodyType): Code that runs whenever try block is entered.
  • function (functionType): Represents a function definition.
  • get (accessorFunctionWithVisibilityType): A getter function, used with a property.
  • goto (gotoType): A goto to a named label.
  • gotoCase (gotoCaseType): A goto to another case. Used inside switch statements.
  • implementsInterface (implementsInterfaceType): The class, structure, or interface implements the specified interface. The derivesFromClass element should be used to specify the primary non-interface base class of a class element.
  • increment (incrementType): Increment the expression by one. The expression will generally be a nameRef or a call* element with type='field'.
  • initialize (expressionContainerType): An initializer for a local, field, iterator, autoDispose, constructor, or lock element.
  • initializeLoop (initializeLoopType): A statement, or a local to initialize before a loop is run for the first time.
  • inlineStatement (inlineStatementType): Statements and complex expressions that have return types. The data type is used for declaring formatter-generated local variables for languages that do not natively support these constructs.
  • interface (interfaceType): An interface definition. Contains procedure definitions. (C#:interface, VB:Interface)
  • interfaceMember (interfaceMemberType): This procedures implements the specified member of the interface. The interface should correspond to an implementsInterface element on the containing type. Note that the PLiX formatters will automatically take care of the cases where the names are different, or the other member settings are no compatible with direct interface implementation (static, private, etc). This is a required element for all interface members, even if they have public visibility and the same name.
  • iterator (iteratorType): Extends a simple type reference by allowing parametrization of the type.
  • label (labelType): A named label. The name can be referenced from a goto element.
  • leadingInfo (leadingInfoType): Non-code elements immediately preceding a member, type, or namespace. These are defined as child elements so they are clearly associated. Leading info is often accompanied by trailingInfo.
  • left (expressionContainerType): The left hand side of binaryOperator, conditionalOperator, nullFallbackOperator, or assign elements.
  • local (localType): An element representing a local variable
  • lock (lockType): A lock construct for cross-thread synchronization.
  • loop (loopType): A looping construct. Use iterator for set iteration. (C#: for, do, while; VB: For, Do, While)
  • nameRef (nameRefType): Use of a named local, parameter, or named parameter.
  • namespace (namespaceType): A namespace definition. Contains other namespace definitions and type definitions. (C#:namespace, VB:Namespace)
  • namespaceImport (namespaceImportType): A namespace import into the file. (C#:using, VB:Imports)
  • nullFallbackOperator (nullFallbackOperatorType): A null fallback operator with left and right elements. If the left expression is null, then the right expression is used. Only valid inside an inlineStatement element.
  • nullKeyword (keywordType): Keyword representing a null object value. (C#: null, VB: Nothing)
  • onAdd (accessorFunctionType): A custom function called when a listener is attached to an event.
  • onFire (accessorFunctionType): A custom function called when an event is raised.
  • onRemove (accessorFunctionType): A custom function called when a listener is detached from an event.
  • operatorFunction (operatorFunctionType): Represents an operator function definition.
  • param (paramType): A parameter definition. Use returns to define a return value.
  • parametrizedDataTypeQualifier (parametrizedDataTypeQualifierType): Allows data type qualifiers to include parametrization. Used in place of @dataTypeQualifier. Ignored if an intrinsic type is specified.
  • passMemberTypeParam (passMemberTypeParamType): Represents a type parameter passed to a function, property, or event.
  • passParam (passParamType): Pass data to a parameter. Contains an expression.
  • passTypeParam (passTypeParamType): Represents a type parameter passed to a type reference.
  • pragma (pragmaType): A code pragma element. Used for regioning, conditional compilation, and other pragma types.
  • property (propertyType): Represents a property definition.
  • return (optionalExpressionBodyStatementType): Represents a return statement.
  • returns (returnsType): The definition of a return value for a procedure.
  • right (expressionContainerType): The right hand side of binaryOperator, conditionalOperator, nullFallbackOperator, or assign elements.
  • root : The root PLiX element. Contains namespaceImport and namespace elements.
    • namespaceImport (namespaceImportType) (optional, repeat): A namespace import into the file. (C#:using, VB:Imports)
    • namespace (namespaceType) (optional, repeat): A namespace definition. Contains other namespace definitions and type definitions. (C#:namespace, VB:Namespace)
  • set (accessorFunctionWithVisibilityType): A setter function, used with a property.
  • string (stringType): A string literal. Nested string elements are automatically concatenated by plix formatters.
  • structure (structureType): A structure definition. Contains nested type definitions, fields, and procedure definitions. (C#:struct, VB:Structure)
  • switch (switchType): Represents a switch block. (C#: switch, VB: Select Case)
  • thisKeyword (keywordType): Keyword representing the this object. (C#: this, VB: Me)
  • throw (optionalExpressionBodyStatementType): Represents a throw statement.
  • trailingInfo (trailingInfoType): Non-code elements immediately after a member, type, or namespace. These are defined as child elements instead of siblings to enable reordering of member elements. Trailing info is often accompanied by leadingInfo and should balance any open leading pragma elements.
  • trueKeyword (keywordType): Keyword representing the boolean true value.
  • try (tryType): A try construct. Contains body, catch, alternate, and finally blocks.
  • typeConstraint (typeParamConstraintType): A type reference indicating required class and interface support for a typeParam. Note that TypeParam attributes are used to specify standard constraints (default constructor, reference type, value type)
  • typeOf (typeOfType): A direct type reference expression. (C#: typeof, VB: GetType)
  • typeParam (typeParamType): A type parameter definition on a type or member (also known as a formal parameter). Contained by class, interface, structure, delegate, function, property, and event definitions.
  • unaryOperator (unaryOperatorType): A unary operator.
  • value (valueType): A primitive value. The type and data of the value must be specified.
  • valueKeyword (keywordType): Keyword representing the language-specific name used to reference a setter value. (C#: value, VB: not a keyword, but Value is the standard usage)
